# Settlement Designer

Editor dock for quickly blocking out settlement scenes.

## Workflow

1. Open or create a settlement scene.
2. The edited 3D scene root is used as the settlement root by default. Click **Set Up Scene Root** to attach the `Settlement` script and generate the expected node tree.
3. Click **Load Grid Settings** to copy spacing and orientation from the selected `HexGrid` or the edited scene.
4. The setup creates one `footprint_0_0` hex. Select it in the 3D scene to show six editor-only expansion handles around it.
5. Click a worldspace handle to grow the settlement into any shape.
6. The selected footprint readout shows the active footprint mesh.
7. Use **Add Boundary Walls** to trace lightweight wall placeholders around the current footprint.
8. Select wall segments and use **Replace Selected Wall With Entrance** to turn them into entrance scenes.
9. Use **Replace Wall Placeholders** when you are ready to swap remaining placeholders for the real wall scene, inspect the result, then undo or save.
10. Use the **Structures** tab to add settlement pieces or buildable service stages.
11. Use **Remove Footprint Ground** or **Save Without Footprint Ground** before committing a settlement prefab; runtime terrain is generated by the grid, not saved as part of the settlement.

The tool writes into the existing `structures`, `walls`, `entrances`, and `player_spawn` layout expected by `Settlement`, plus a `footprint` authoring container.
The footprint ground is a temporary authoring aid made of single-node mesh markers. The worldspace expansion handles are transient editor controls and are not owned by, or saved into, the settlement scene.
